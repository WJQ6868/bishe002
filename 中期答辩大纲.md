# AI赋能的高校教务系统中期答辩大纲（AI实现逻辑重点版）

## 一、课题定位与中期汇报主线

本课题聚焦“AI赋能高校教务系统”，核心不是重复建设传统教务功能，而是把AI真正嵌入教务服务场景，形成“可配置、可落地、可持续优化”的智能服务能力。  
中期答辩建议围绕一条主线展开：**AI如何从配置端进入业务端，再通过数据回流形成优化闭环**。  
因此，本次汇报重点应放在三方面：  
1）AI能力架构与运行逻辑；  
2）AI客服、AI课程助手、智能教案三大场景实现机制；  
3）可用性验证、当前问题与后续优化计划。

---

## 二、AI总体实现方案（从“能用”到“可管可控”）

当前系统的AI能力不是单点聊天，而是“配置驱动+知识驱动+场景驱动”的组合方案，整体可分为四层：

### 1. AI配置编排层（管理端）

管理员可完成模型配置、知识库管理、工作流绑定、客服策略配置等操作。  
这意味着AI行为不依赖写死代码，而是通过后台配置完成切换和控制，满足中后期扩展需求。

### 2. 知识组织层（知识库）

系统支持文档上传、文本抽取、分块存储、检索召回。  
核心价值是把“学校制度、课程资料、教学文档”转化为AI可检索上下文，降低纯大模型幻觉风险。

### 3. 推理执行层（模型调用）

系统按业务请求动态选择模型并发起推理，支持流式输出。  
在同一套机制下可服务客服问答、课程答疑、教案生成，保证能力复用和一致性。

### 4. 业务接入层（师生使用场景）

AI客服、AI课程助手、智能教案分别对接学生端、教师端、管理员端，形成不同角色可见、可操作、可反馈的业务入口。

## 三、三大AI功能的实现逻辑

## 1）AI客服：从“问答”到“可运营服务”

AI客服的设计目标是处理高频教务咨询，降低人工重复回复压力。当前实现逻辑为：

1. 管理员配置客服工作流（欢迎语、推荐问题、提示词、模型、知识库）。  
2. 用户提问后，系统先定位客服工作流并确定可用模型。  
3. 系统在客服知识库中检索相关片段，组装上下文。  
4. 模型基于“用户问题+知识上下文”生成回答并流式返回。  
5. 若模型不可用或未配置，系统返回明确提示，避免无响应。  

与普通聊天相比，这套机制的特点是：**可配置、可追溯、可兜底**。  
中期阶段已证明该链路可运行，具备演示稳定性。

## 2）AI课程助手：从“通用问答”到“课程场景问答”

AI课程助手的目标是让答疑贴近课程，而不是泛化聊天。实现逻辑为：

1. 教师上传课程相关资料，构建课程维度知识基础。  
2. 教师可基于基础助手创建个人/课程专属工作流。  
3. 学生进入课程助手后按课程或工作流提问。  
4. 系统优先使用课程关联知识片段进行检索增强，再生成回答。  

该方案实现了“教师供给知识、学生按课提问、AI按课回答”的闭环，避免答非所问，是课题“专业适配”目标的重要落地点。

## 3）智能教案：从“一次生成”到“任务化生成”

智能教案不是简单调用模型返回一段文本，而是采用任务机制：

1. 教师创建教案任务（课程、标题、提纲）。  
2. 系统绑定教案工作流，装配模型与知识来源。  
3. 生成过程按流式输出，任务状态实时变化。  
4. 生成结果回写任务记录，支持后续查看和复用。  

这种任务化设计有两个价值：  
- 可管理：每次生成都有状态和结果留存；  
- 可优化：后续可基于任务历史做模板增强和质量评估。

## 四、AI关键运行机制（答辩应重点解释）

### 1. 模型选择优先级机制

系统采用分级策略选择模型：优先用户或场景指定模型，其次工作流绑定模型，最后默认模型兜底。  
该机制保证灵活性与稳定性并存，避免因单一配置异常导致全局不可用。

### 2. 知识库检索增强机制

文档会被拆分为可检索片段，提问时先召回相关片段，再把上下文送给模型。  
这一机制决定了AI回答是否“贴近校内规则和课程内容”，是系统区别于普通聊天机器人的关键。

### 3. 工作流绑定机制

每个AI场景都可绑定不同“模型+知识库+策略参数”。  
这使系统可以实现“客服偏稳健、课程助手偏专业、教案偏结构化”的差异化行为，而不是一个模型处理所有问题。

### 4. 运行日志与反馈机制

系统记录AI调用日志，可按功能和角色查看使用情况。  
中期阶段已有实际日志数据，说明AI能力已进入真实使用链路，而非静态展示。

## 六、存在问题与对应措施（AI视角）

### 1. 当前问题

1）模型能力依赖第三方接口，可能受额度、网络、延迟波动影响；  
2）知识库质量决定回答质量，部分场景知识覆盖仍可加强；  
3）AI输出评价体系仍偏人工判断，自动化质量评估不足。

### 2. 解决措施

1）维持多模型可切换与默认兜底，答辩前执行连通性检查；  
2）继续补充高频问答与课程核心资料，提升检索命中率；  
3）增加失败重试、结果评分与人工复核机制，形成可量化优化闭环。
